The DeviaTDD governance toolkit lacked CLI commands for constitution validation, issue inspection, and tasks inspection — forcing developers to read raw JSONL files to understand project state. This adds three new CLI surfaces (`deviate constitution`, `deviate issues list`, `deviate tasks list`) with Rich table rendering, JSON output, and orphan claim detection for SPECIFIED issues. It also audits and patches the constitution seed template to guarantee all 6 required `${VARIABLE}` placeholders are present.

Constitution CLI: `deviate constitution pre` validates `specs/constitution.md` and extracts test/lint/typecheck commands into a JSON contract; `deviate constitution post <manifest>` validates section structure and commits. Both handle missing files and missing sections with structured `FAILURE` responses.

Inspection CLI: `deviate issues list` parses `specs/issues.jsonl` bottom-up with `--type`, `--status`, and `--json` filters, and detects orphan claims on SPECIFIED issues via `git ls-remote --heads` (three-state outcome: `true`, `false`, `null`). `deviate tasks list` parses the active issue's `tasks.jsonl` with status derivation. Both commands render Rich tables by default and emit valid JSON arrays with `--json`.

Seed audit: `validate_placeholders()` in `core/constitution.py` audits all 6 `${VARIABLE}` tokens in `constitution_seed.md` (PROJECT_NAME, REPO_ROOT, TARGET_BACKEND_FRAMEWORK, TARGET_PACKAGE_MANAGER, TARGET_TEST_RUNNER, TARGET_COVERAGE_MINIMUM), returning a `PlaceholderAuditResult` with missing variable detection. The seed template was patched with the 4 missing placeholders.
