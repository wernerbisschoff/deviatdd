## Discovery Audit Results

### TUI-Focused Headline Findings

1. **The TUI renderer (C6) is greenfield.** `specs/_product/release-next.md:48` AC §1 declares `src/deviate/tui/renderer.py` as an install deliverable. `ls src/deviate/` returns `cli core main.py prompts state ui visual` only — neither `rpc/` nor `tui/` exists on disk as of this scan (`src/deviate/` listing captured at extraction time).
2. **The Rich UI baseline is mature and stable.** `src/deviate/ui/render.py` (1.6 KB) and `src/deviate/ui/pipeline.py` (15.0 KB) and `src/deviate/ui/monitor.py` (7.0 KB) all exist and are referenced from the meso/micro CLI entry points.
3. **`stdout_lock` is the thread-safety seam every redraw must take.** `src/deviate/ui/render.py:24-30` defines a module-level `threading.Lock` because a historical SIGSEGV path was triggered when background reader threads raced main-thread `Console.print` writes on macOS.
4. **`is_interactive()` gates Rich region behavior.** `src/deviate/ui/render.py:40-46` reads `CI` env var + `os.isatty(sys.stdout.fileno())`. CI / non-TTY output must skip the live region and route to final-summary (`specs/_product/flows/flows-streaming.md` Happy path step 7 + Metric "TUI shows live progress without growing past 10 lines").
5. **The architecture anchor is concrete enough to scaffold against.** The `TuiRenderer` entity at `specs/_product/domain-model.md:86-92` pins `max_lines: int (default 10)` and the contract surface at `specs/_product/architecture.md:62-66` pins `feed(event)` + `flush_summary(text)` — both directly translatable to a `Renderer` class signature.
6. **The existing `OrchestrationMonitor` is the integration candidate.** `src/deviate/ui/monitor.py` already maintains an in-process event deque via `OrchestrationMonitor.push_event(event_type, **data)`. C5 → C6 can subscribe through this surface or be sibling-wired; the exact seam is a micro-phase decision (per the architectural discovery-only discipline, this is observed, not recommended).
7. **`AgentConfig.pi_rpc` already exists but is not wired into any command.** `src/deviate/state/config.py:18-21` declares `pi_rpc: bool = False`. No `.deviate/config.toml` instance in the repo sets it (only `[agent].backend = "omp"` at `.deviate/config.toml`). The flag is unreferenced from `src/deviate/cli/`. This is the opt-in point I04.1 + I04.2 will widen (`specs/_product/release-next.md:39-40`).

## Architectural Baselines

[Pattern_Over_Instance]: One representative example per category. All paths strictly relative to `repo_root`. TUI-focused slice of FLOW-04.

- **Existing Architectural Patterns**:
  - **Contract-surface declaration in `architecture.md`, implementation deferred to RED/GREEN** (`specs/_product/architecture.md:62-66`, C6): "**Contract surface:** `Renderer(max_lines: int = 10) -> Renderer`; `renderer.feed(event: Event) -> None`; `renderer.flush_summary(text: str) -> None`. **Owns:** terminal cursor state, the fixed redraw region, the final-summary region. **Implementation:** Rich terminal primitives (already mandated by `specs/constitution.md:23`); exact class selection deferred to RED/GREEN since Rich is not available in libref for verification."   - **`stdout_lock` as the in-process thread-safety seam** (`src/deviate/ui/render.py:24-30`): A module-level `threading.Lock` named `stdout_lock` serializes every write to the interactive `Console` / raw `sys.stdout` fd.
    ```python
    _stdout_lock = threading.Lock()
    stdout_lock = _stdout_lock
    ```
    The comment block in the same file explains the lock closes a historical SIGSEGV path in `run --all` JUDGE on macOS when background reader threads race main-thread `c.print` writes. Any new TUI renderer MUST take this lock on every redraw. The `str_pyobject`-style `with stdout_lock:` pattern is the established idiom in this codebase.
  - **`is_interactive()` gates Rich-region behavior** (`src/deviate/ui/render.py:40-46`): Reads `os.environ.get("CI")` and `os.isatty(sys.stdout.fileno())`. The TUI region needs to be skipped when stdout is not a TTY (CI / pipes) and the final-summary region takes over.
  - **`OrchestrationMonitor` event-queue pattern** (`src/deviate/ui/monitor.py`): `OrchestrationMonitor` already maintains an in-process event deque with `push_event(event_type, **data)` and a `VALID_EVENT_TYPES` allow-list. The new TUI renderer can subscribe to or compose with this monitor without re-architecting the orchestration layer. The exact C5 → C6 seam is a micro-phase decision; this explore records the seam exists.

## Ecosystem Research

[Web_Discovery]: Catalog from primary sources via `libref`. The `pi@latest` and `oh-my-pi@latest` packages are already registered in the local libref registry (`libref list` output: `pi@latest 1.1 MB / 454 sections`, `oh-my-pi@latest 2.8 MB / 1337 sections`).

- **Best Practices**:
  - **Strict-LF JSONL framing is mandatory.** `pi@latest` (`packages/coding-agent/docs/rpc.md`): "RPC mode uses strict JSONL semantics with LF (`\n`) as the only record delimiter. … In particular, Node `readline` is not protocol-compliant for RPC mode because it allows Unicode separators as newlines." `oh-my-pi@latest` (`docs/rpc.md`) confirms the same protocol: "newline-delimited JSON protocol over stdio."
  - **Subprocess isolation over in-process linking.** `pi@latest` (`packages/coding-agent/README.md` SDK / RPC Mode section): "For non-Node.js integrations, use RPC mode over stdin/stdout: `pi --mode rpc`." `deviate` is Python driving a TypeScript runtime, and process isolation is mandated by the constitution §1 Git Isolation Principle.
  - **Request/response correlation by generated id.** `oh-my-pi@latest` (`docs/rpc.md`, RPC Command Schema section): `RpcCommand` is `{"id?", "type", ...}` and responses carry the same `id` for correlation. C5 → C6 sits downstream of C4's correlation; the TUI renderer does not own correlation but receives already-categorized `Event` instances.
  - **Streaming deltas surface as sub-events of `message_update`.** `oh-my-pi@latest` (`docs/rpc.md`, Event Stream Schema section): "`message_update` includes streaming deltas in `assistantMessageEvent` (text/thinking/toolcall deltas)." The Product-layer `StreamingDelta` entity at `specs/_product/domain-model.md:80-83` enumerates the three delta kinds directly.

- **Common Use Cases & Pitfalls**:
  - **Pitfall — Unicode line separators.** `pi@latest` (`packages/coding-agent/docs/rpc.md`): "Do not use generic line readers that treat Unicode separators as newlines." Release-next AC §6 mandates a regression test that feeds records containing `\u2028` and `\u2029` and asserts the splitter does NOT treat them as record terminators.
  - **Pitfall — Backwards compatibility.** `specs/_product/release-next.md:21`: "legacy logging path preserved when C2–C6 are absent." C6 must be opt-in; existing print-mode invocations of `pi`/`omp`/`opencode`/`claude`/`droid` must keep working (legacy stdout reader at `src/deviate/core/agent.py:215-272`).
  - **Pitfall — Redraw latency budget.** `specs/_product/flows/flows-streaming.md` Metrics: "`tui_redraw_latency_ms` p95 ≤ 100." Any chosen Rich primitive (`Live`, `Layout`, `Group`) must hit this ceiling; the regression test under AC §4 drives 1,000 events and asserts ≤ 10 lines — implicit redraw-latency assertion is achievable through wall-clock timing of the test.
  - **Pitfall — TTY detection and CI.** `specs/constitution.md:54-56` mandates E2E via `bats tests/e2e/`. E2E against a TUI region cannot be captured from a non-TTY runner; the renderer must fall back to final-summary when `is_interactive()` is False.

- **Standard Tooling**:
  - **Available offline docs.** `libref list` confirms `pi@latest` (1.1 MB, 454 sections) and `oh-my-pi@latest` (2.8 MB, 1337 sections) are registered. Both packages document RPC mode at `packages/coding-agent/docs/rpc.md` (Pi) and `docs/rpc.md` (OH-MY-PI) — same protocol vocabulary. No `rich` entry in the libref registry; GREEN phase must consult Rich's own source if needed.
  - **No new Python dependency required.** The wire format and subprocess boundary are both reachable via stdlib (`subprocess`, `json`, `threading`). `rich>=13.0` (already in `pyproject.toml:32`) is the only TUI primitive required; no need for `prompt_toolkit`, `textual`, or `urwid`.
  - **Test boundary.** Tests can use `subprocess.Popen` against a stub script that writes a recorded JSONL stream to stdout, exercising C3 framing + C5 event adapter without spawning a real `pi`/`omp` binary. The TUI redraw-budget test (AC §4) does NOT need a subprocess at all — it injects events directly into `Renderer.feed`.
