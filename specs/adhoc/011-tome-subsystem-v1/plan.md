## Data Flow Analysis
1. **Skill Template Creation**: Author writes seven `SKILL.md` files → filesystem at `src/deviate/prompts/skills/<name>/SKILL.md` → `discover_skills()` at `src/deviate/core/skills.py:20-26` enumerates the directories and includes the seven new names in its sorted return.
2. **Agent Skill Installation**: `deviate setup --agent claude` runs → `_install_skills_to_agents` at `src/deviate/cli/__init__.py:522-535` calls `discover_skills()` (now 31 names) → for each name, `install_skill(name, target_dir, workdir)` at `src/deviate/core/skills.py:114-146` reads the source `SKILL.md`, composes the body with `core.md` + `{layer}-skill.md` (no `tome-skill.md` exists in `src/deviate/prompts/core/`, so layer prefix is a no-op), and writes the result to `workdir/.claude/skills/<name>/SKILL.md`. The install is idempotent: byte-equal existing content returns `False` and emits a `SKIP` log line.
3. **Agent Invocation (out of v1 scope)**: Developer runs `/tome-classify` against a target repo → agent reads `workdir/.claude/skills/tome-classify/SKILL.md` → follows the inlined classifier prompt → reads commit/diff from target repo → emits classification report → developer runs the relevant writer skill (e.g. `/tome-write-tutorial`) → writer reads its inlined prompt → writes markdown to `apps/docs/src/content/docs/<quadrant>/` in the *target repo* (not in DeviaTDD's repo, per `specs/_product/architecture.md:18`).
4. **Verifier Loop (out of v1 scope)**: Developer runs `/tome-verify-docs` → verifier reads classification report + updated docs + commit diff → emits human-readable PASS/FAIL/boundary-violation report → no auto-routing back to writers (per `specs/_product/architecture.md:20`).
