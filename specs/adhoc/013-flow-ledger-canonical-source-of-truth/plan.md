## Data Flow Analysis
1. **Input layer**: `specs/_product/flows/index.md` (markdown table) + `specs/issues.jsonl::IssueRecord.flow_refs` + existing `specs/_product/flows.jsonl` rows (if any).
2. **Seed step**: `_parse_flows_index` → `list[FlowRecord]` → `append_flow_record` (idempotent on `flow_id`) → `append_flow_event` × 2 per flow (`FLOW_DISCOVERED` + `FLOW_DOCUMENTED`, idempotent on compound key).
3. **Reverse-index step**: load `specs/issues.jsonl` via `_read_ledger` (line 40) → for each `IssueRecord`, for each `flow_refs` token, resolve the canonical flow_id via `FlowRecord` lookup → if missing, log `ORPHANED_FLOW_REF` warning; if present, `append_flow_event` with `event_issue_id=record.issue_id`, `event_type="FLOW_REFERENCED_BY_ISSUE"`, `timestamp=record.created_at`.
4. **Derive step**: `load_flow_coverage` parses `flows/index.md` (canonical set) → reads `flows.jsonl` row-by-row → replays events → for each flow computes:
   - `discovered_status` = `"DISCOVERED"` iff any `FLOW_DISCOVERED` event exists for the flow.
   - `doc_status` = `"DOCUMENTED"` iff any `FLOW_DOCUMENTED` event exists.
   - `impl_status` = `"CONFIRMED_IMPLEMENTED"` iff any `FLOW_CONFIRMED_IMPLEMENTED` event exists; `"PARTIALLY_IMPLEMENTED"` iff `FLOW_IMPLEMENTATION_EVIDENCE_ADDED` exists; else `"UNCONFIRMED"`.
   - `drift_flag` derived via taxonomy: `(UNDISCOVERED, _, _)` → `PROMPT_ONLY_NO_CODE`; `(DISCOVERED, UNDOCUMENTED, _)` → `DOC_ARTIFACT_ONLY`; `(DISCOVERED, DOCUMENTED, UNCONFIRMED)` → `DOCUMENTED_BUT_NOT_IMPLEMENTED`; `(DISCOVERED, DOCUMENTED, CONFIRMED_IMPLEMENTED)` with no `last_referenced_by_issue_id` → `IMPLEMENTED_BUT_UNDOCUMENTED`; flow in `flows.jsonl` but absent from `flows/index.md` → `ORPHANED_FLOW`; `_deprecated` status from `FlowRecord` with no `FLOW_DEPRECATED` event → `STALE_DRIFT`; all checks pass → `OK`.
5. **Render step**: Rich `Table` with six columns. Drift rows highlighted in yellow. Output to stdout via `console.print`.
6. **Storage**: `specs/_product/flows.jsonl` is the canonical append-only ledger. `FlowCoverage` rows are never persisted — they are derived per-call from the JSONL + markdown + issues ledger per §1 *Append-Only Ledger Protocol* (constitution v0.7.0).
